# 使用手册

这是一份中文使用说明，包含：

- 如何在 **Claude Code**、**OpenAI Codex CLI**、**GitHub Copilot** 三个平台上安装和使用
- 如何使用 `/ssh-remote`
- 如何使用 `/ssh-remote-hook-on` 和 `/ssh-remote-hook-off`
- 如何开启/关闭 hook 模式
- hook 模式会强制哪些 Bash 命令走远程 SSH

仓库地址：

- `https://github.com/Ivanbeethoven/claude-ssh-remote`

---

## 一、最快使用方式

### Claude Code

```bash
git clone https://github.com/Ivanbeethoven/claude-ssh-remote.git
cd claude-ssh-remote
cc --plugin-dir .
```

启动 Claude Code 后：

```text
/ssh-remote azure
```

### OpenAI Codex CLI

```bash
git clone https://github.com/Ivanbeethoven/claude-ssh-remote.git
```

把 `codex/skills/` 下的 skill 注册到 `~/.codex/config.toml`（详见 `codex/config-snippet.toml`）。

启动 Codex 后：

```text
$ssh-remote azure
```

### GitHub Copilot

把指令文件复制到你的项目：

```bash
cp github/copilot-instructions.md .github/copilot-instructions.md
cp -r github/instructions/ .github/instructions/
cp AGENTS.md AGENTS.md
```

然后在 Copilot 里正常对话即可。

## 二、SSH 配置

无论哪个平台，都需要先确保本机终端能通过 SSH 配置连接：

```bash
ssh azure
```

示例 `~/.ssh/config`：

```sshconfig
Host azure
  HostName 20.1.2.3
  User hxy
  Port 22
  IdentityFile ~/.ssh/id_rsa
```

## 三、skill 的基本用法

### 1. 只连接远程服务器

| 平台 | 命令 |
|------|------|
| Claude Code | `/ssh-remote azure` |
| Codex CLI | `$ssh-remote azure` |
| Copilot | 直接让 agent 执行 `ssh azure` |

### 2. 连接后进入远程目录

| 平台 | 命令 |
|------|------|
| Claude Code | `/ssh-remote azure /srv/my-app` |
| Codex CLI | `$ssh-remote azure /srv/my-app` |

### 3. 连接后进入持久会话

| 平台 | 命令 |
|------|------|
| Claude Code | `/ssh-remote azure /srv/my-app dev` |
| Codex CLI | `$ssh-remote azure /srv/my-app dev` |

持久会话模式：

- 优先使用 `tmux`
- 没有 `tmux` 时退回 `screen`
- 都没有时退回普通 shell

## 五、hook 模式是什么

hook 模式是"强制远程优先"的版本。

开启后，它会拦截 `Bash` 工具。

当前版本只针对 **Bash**，不会自动处理所有工具类型。

它的核心行为是：

- 对安全、非交互式 Bash，优先改写成 `ssh <host> '原命令'`
- 在远程服务器执行
- 原本的本地 Bash 会被阻止
- 对明显交互式或不适合透明转发的命令，不自动执行，只阻止并提示你该怎么在远程运行

适合你的场景：

- 希望尽量不要在本机执行 Bash
- 希望命令统一走某一台远程机器
- 希望整个开发过程以远程环境为主

## 六、三个平台的 hook 能力对比

| 特性 | Claude Code | Codex CLI | GitHub Copilot |
|------|------------|-----------|----------------|
| 程序级 hook 拦截 | 有 (PreToolUse) | 有 (实验性) | 无 |
| skill 定义 | SKILL.md | SKILL.md | 无 |
| 指令文件 | — | AGENTS.md | copilot-instructions.md |
| 执行保证 | 强 (程序拦截) | 强 (需开启特性) | 弱 (靠指令遵循) |
| 配置文件格式 | YAML frontmatter | TOML | YAML frontmatter |
| 配置文件路径 | `.claude/claude-ssh-remote.local.md` | `.codex/ssh-remote.local.toml` | `.claude/claude-ssh-remote.local.md` |

**重要提示：** Copilot 没有程序级 hook，只能通过指令文件告诉 agent 应该怎么做。
agent 可能偶尔不遵循指令。如果需要严格保证远程执行，建议用 Claude Code 或 Codex CLI。

## 七、hook 会拦什么

当前只拦 `Bash` 工具。

### 会直接放行的命令

以下命令默认不会被重复改写：

- `ssh ...`
- `scp ...`
- `sftp ...`

因为它们本身已经是远程相关命令。

### 会尝试自动走远程的命令

例如这类普通 Bash：

```bash
pwd
ls
uname -a
python --version
```

hook 会尽量改成：

```bash
ssh azure 'pwd'
ssh azure 'ls'
```

然后阻止原本地 Bash。

### 会被阻断、但不自动代跑的命令

例如明显交互式或不适合透明代理的命令：

- `vim`
- `nano`
- `top`
- `htop`
- `tmux`
- `screen`
- `tail -f`
- 带明显复杂管道/链式控制的 Bash

这些命令通常会被拦下，并提示你：

```bash
ssh azure '原命令'
```

你再自己决定是否在远程执行。

## 八、配置文件格式

### Claude / Copilot（YAML frontmatter）

文件：`.claude/claude-ssh-remote.local.md`

```markdown
---
enabled: true
ssh_host: azure
mode: rewrite_or_block
---
```

### Codex（TOML）

文件：`.codex/ssh-remote.local.toml`

```toml
enabled = true
ssh_host = "azure"
mode = "rewrite_or_block"
```

### 字段说明

- `enabled: true` — 开启 hook
- `ssh_host: azure` — 指定强制走的 SSH config 名称
- `mode: rewrite_or_block` — 先尝试自动改写并远程执行，如果失败，就阻止本地执行
- `mode: block_only` — 不自动执行，只阻止并提示你改成 `ssh azure '...'`

---

## 九、如何手动开启/关闭 hook 模式

### 手动开启

创建对应的配置文件并写入内容（见上一节）。

### 手动关闭

- 改 `enabled: true` 为 `enabled: false`
- 或直接删除配置文件
- 没有配置文件时，hook 不会生效

---

## 十、建议的使用顺序

1. 先确保本机终端可用：

```bash
ssh azure
```

2. 启动对应的 AI 工具

3. 先用命令入口测试：

```text
/ssh-remote azure    # Claude Code
$ssh-remote azure    # Codex CLI
```

4. 如果你希望后续 Bash 全走远程，再执行：

```text
/ssh-remote-hook-on azure    # Claude Code
$ssh-remote-hook-on azure    # Codex CLI
```

5. 如果用 Copilot，复制指令文件到项目后，直接对话即可

---

## 十一、当前限制

- 主要针对 `Bash` 工具
- 不会自动禁止所有本地文件读写
- 更适合作为"命令执行强制远程"的稳定版
- Copilot 的远程执行只能靠指令约束，无法程序级保证
