# 使用手册

这是一份中文使用说明，包含：

- 如何安装这个 plugin
- 如何使用 `/ssh-remote`
- 如何使用 `/ssh-remote-hook-on` 和 `/ssh-remote-hook-off`
- 如何开启/关闭 hook 模式
- hook 模式会强制哪些 Bash 命令走远程 SSH

仓库地址：

- `https://github.com/Ivanbeethoven/claude-ssh-remote`

## 一、最快使用方式

```bash
git clone https://github.com/Ivanbeethoven/claude-ssh-remote.git
cd claude-ssh-remote
cc --plugin-dir .
```

启动 Claude Code 后，直接使用：

```text
/ssh-remote azure
```

这里的 `azure` 就是你在 `~/.ssh/config` 里写好的 Host 名称。

例如：

```sshconfig
Host azure
  HostName 20.1.2.3
  User hxy
  Port 22
  IdentityFile ~/.ssh/id_rsa
```

只要你本机终端里这条命令能通：

```bash
ssh azure
```

那么 plugin 里就可以直接这样用：

```text
/ssh-remote azure
```

---

## 二、skill 的基本用法

### 1. 只连接远程服务器

```text
/ssh-remote azure
```

等价于：

```bash
ssh azure
```

### 2. 连接后进入远程目录

```text
/ssh-remote azure /srv/my-app
```

### 3. 连接后进入持久会话

```text
/ssh-remote azure /srv/my-app dev
```

这个模式下：

- 优先使用 `tmux`
- 没有 `tmux` 时退回 `screen`
- 都没有时退回普通 shell

## 三、快速开关 hook

### 开启 hook

```text
/ssh-remote-hook-on azure
```

这个命令会在当前项目写入：

```text
.claude/claude-ssh-remote.local.md
```

默认内容是：

```markdown
---
enabled: true
ssh_host: azure
mode: rewrite_or_block
---
```

### 关闭 hook

```text
/ssh-remote-hook-off
```

这个命令会把配置改成关闭状态。

---

## 四、hook 模式是什么

hook 模式是“强制远程优先”的版本。

开启后，它会拦截 Claude 的 `Bash` 工具。

当前版本只针对 **Bash**，不会自动处理所有工具类型。

它的核心行为是：

- 对安全、非交互式 Bash，优先改写成 `ssh <host> '原命令'`
- 在远程服务器执行
- 原本的本地 Bash 会被阻止
- 对明显交互式或不适合透明转发的命令，不自动执行，只阻止并提示你该怎么在远程运行

适合你的场景：

- 希望 Claude 尽量不要在本机执行 Bash
- 希望命令统一走某一台远程机器
- 希望整个开发过程以远程环境为主

---

## 五、hook 会拦什么

当前只拦 Claude 的 `Bash` 工具。

### 会直接放行的命令

以下命令默认不会被重复改写：

- `ssh ...`
- `scp ...`
- `sftp ...`

因为它们本身已经是远程相关命令。

### 会尝试自动走远程的命令

例如这类普通 Bash：

```bash
pwd
ls
uname -a
python --version
```

hook 会尽量改成：

```bash
ssh azure 'pwd'
ssh azure 'ls'
```

然后阻止原本地 Bash。

### 会被阻断、但不自动代跑的命令

例如明显交互式或不适合透明代理的命令：

- `vim`
- `nano`
- `top`
- `htop`
- `tmux`
- `screen`
- `tail -f`
- 带明显复杂管道/链式控制的 Bash

这些命令通常会被拦下，并提示你：

```bash
ssh azure '原命令'
```

你再自己决定是否在远程执行。

## 六、如何手动开启 hook 模式

如果你不想通过命令开启，也可以手动创建文件：

```text
.claude/claude-ssh-remote.local.md
```

写入：

```markdown
---
enabled: true
ssh_host: azure
mode: rewrite_or_block
---
```

字段说明：

- `enabled: true`
  - 开启 hook
- `ssh_host: azure`
  - 指定强制走的 SSH config 名称
- `mode: rewrite_or_block`
  - 先尝试自动改写并远程执行
  - 如果失败，就阻止本地执行

### 推荐配置

```markdown
---
enabled: true
ssh_host: azure
mode: rewrite_or_block
---
```

---

## 七、如何手动关闭 hook 模式

有两种方法。

### 方法 1：用命令关闭

```text
/ssh-remote-hook-off
```

### 方法 2：直接改配置

```markdown
---
enabled: false
ssh_host: azure
mode: rewrite_or_block
---
```

### 方法 3：删除配置文件

直接删除：

```text
.claude/claude-ssh-remote.local.md
```

没有这个文件时，hook 不会生效。

---

## 八、两种 hook 工作模式

### 1. rewrite_or_block

配置：

```markdown
---
enabled: true
ssh_host: azure
mode: rewrite_or_block
---
```

行为：

- Claude 想执行本地 Bash
- 如果命令安全且适合远程透明转发，就改写成：

```bash
ssh azure '原命令'
```

- 远程执行成功后，阻止本地 Bash
- 如果命令不适合透明转发，就只阻止并提示远程命令

这是默认推荐模式。

### 2. block_only

配置：

```markdown
---
enabled: true
ssh_host: azure
mode: block_only
---
```

行为：

- 不自动执行远程命令
- 只阻止本地 Bash
- 并提示你应该改成：

```bash
ssh azure '原命令'
```

适合你想完全人工确认远程命令的时候。

---

## 九、建议的使用顺序

建议你这样用：

1. 先确保本机终端可用：

```bash
ssh azure
```

2. 再启动 plugin：

```bash
cc --plugin-dir .
```

3. 先用命令入口测试：

```text
/ssh-remote azure
```

4. 如果你希望 Claude 后续 Bash 全走远程，再执行：

```text
/ssh-remote-hook-on azure
```

这样最稳。

## 十、当前限制

当前 hook 版本的能力边界：

- 主要针对 Claude 的 `Bash` 工具
- 不会自动禁止所有本地文件读写
- 更适合作为“命令执行强制远程”的稳定版

4. **根据不同项目目录切不同远程 host**
